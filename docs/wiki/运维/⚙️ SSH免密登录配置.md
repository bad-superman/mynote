配置 SSH 免密登录可以让你更安全、便捷地连接远程服务器，核心原理是**使用密钥对认证代替密码验证**。整个过程主要分为三步：**生成密钥对**、**上传公钥**和**验证登录**。

下面是最常用的标准配置方法，适用于大多数 Linux、macOS 和 Windows 环境：

### ⚙️ 操作步骤

1.  **生成密钥对 (在客户端机器上执行)** 打开你的终端（Linux/macOS）或命令提示符/PowerShell（Windows），输入以下命令：
    
    ```bash
    ssh-keygen -t rsa -b 4096
    ```
    
    -   `-t rsa` 指定密钥类型为 RSA 。
        
    -   `-b 4096` 指定密钥长度为 4096 位，以提高安全性 。
        
    
    执行命令后，系统会提示你：
    
    -   **保存位置**：直接按 `Enter` 键接受默认路径（通常是 `~/.ssh/id_rsa`）。
        
    -   **设置密码短语 (passphrase)**：这是一个额外的安全层，即使私钥被盗，没有密码短语也无法使用。如果为了自动化脚本或追求极致的便捷，可以直接按 `Enter` 键跳过 。如果只是为了日常使用，建议设置一个。
        
2.  **将公钥复制到远程服务器** 这是最关键的一步，目的是将你本地生成的公钥（`id_rsa.pub`）的内容，添加到远程服务器的信任列表中。
    
    -   **方法一：使用** `ssh-copy-id` **命令 (最推荐，最简单)** 大多数 Linux 和 macOS 系统都自带这个工具。在终端执行：
        
        ```bash
        ssh-copy-id 用户名@远程服务器IP
        ```
        
        例如：`ssh-copy-id user@192.168.1.100` 。 系统会提示你输入远程服务器的用户密码，这是**最后一次**输入密码。验证通过后，公钥就会自动添加到正确的位置 。
        
    -   **方法二：手动复制 (当没有** `ssh-copy-id` **命令时使用，如部分 Windows 环境)**
        
        -   **第一步**：查看并复制本地公钥的内容。
            
            ```bash
            cat ~/.ssh/id_rsa.pub
            ```
            
            选中并复制终端里显示的全部内容 。
            
        -   **第二步**：登录到你的远程服务器。
            
            ```bash
            ssh 用户名@远程服务器IP
            ```
            
            输入密码登录。
            
        -   **第三步**：在远程服务器上，将刚才复制的公钥内容添加到 `~/.ssh/authorized_keys` 文件中，并设置正确的权限。
            
            ```bash
            mkdir -p ~/.ssh
            echo "这里粘贴你刚刚复制的公钥内容" >> ~/.ssh/authorized_keys
            chmod 700 ~/.ssh
            chmod 600 ~/.ssh/authorized_keys
            ```
            
            `chmod` 命令用于设置正确的权限，这是保证 SSH 能够正常工作的关键一步 。
            
3.  **验证免密登录** 现在，可以退出远程服务器，然后在本地终端尝试重新连接：
    
    ```bash
    ssh 用户名@远程服务器IP
    ```
    
    如果一切顺利，你将不再被询问密码，直接登录成功 。如果你在第一步设置了密码短语，这里会提示你输入该短语，而不是服务器密码 。
    

### 🔒 安全加固建议

完成基础配置后，可以考虑以下步骤来提升服务器的安全性：

-   **禁用密码登录**：这是最有效的一步。在确认免密登录可以正常工作后，编辑远程服务器的 SSH 配置文件 `/etc/ssh/sshd_config`，找到或添加以下行：
    
    ```text
    PasswordAuthentication no
    ```
    
    保存文件后，重启 SSH 服务使更改生效：
    
    ```bash
    sudo systemctl restart sshd
    ```
    
    这样，你的服务器将只接受密钥认证，能有效抵御暴力密码破解攻击 。
    

### 💡 进阶技巧

如果你的使用场景更复杂，比如需要管理多个不同用途的服务器（例如工作与个人项目），可以使用 SSH 配置文件进行管理。在本地 `~/.ssh/config` 文件中添加类似下面的配置，就可以用 `ssh work` 这样的简短命令来连接了 。

```text
Host work
    HostName server.example.com
    User workuser
    IdentityFile ~/.ssh/work_rsa
```
